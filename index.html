Install
Up and running in 10 seconds

Three ways to add QA Orchestra to your workflow. Plugin install is recommended — all 10 agents are instantly available in any project.

# 1. Add the marketplace
/plugin marketplace add Anasss/qa-orchestra

# 2. Install the plugin
/plugin install qa-orchestra@qa-orchestra
git clone https://github.com/Anasss/qa-orchestra.git
cp qa-orchestra/.claude/agents/*.md ~/.claude/agents/
git clone https://github.com/Anasss/qa-orchestra.git
cd qa-orchestra
cp examples/CONTEXT.example.md context/CONTEXT.md
claude

After installing, fill in context/CONTEXT.md with your stack details.

Pipeline
From PR to QA report in one command

The orchestrator routes your ticket through specialized agents, each writing structured output for the next.

01

Environment Setup

Checks out the PR branch, starts your app, runs health checks. Works with any stack.

→ environment-status.md
02

Analysis & Design

Parallel: functional reviewer compares diff vs AC while test scenario designer generates scenarios.

→ functional-review.md test-scenarios.md
03

Browser Validation

Navigates the running app via Chrome MCP. Clicks, fills forms, verifies actual behavior.

→ browser-validation.md
04

Reports & Code

Bug reporter files issues. Automation writer generates Playwright/Cypress test code.

→ bug-reports.md automation/*.spec.ts
Agents
10 specialists, one orchestra

Each agent is a standalone Markdown file in .claude/agents/. No framework, no dependencies.

🎴
orchestrator
Sonnet
Routes tickets to the right agents in the right order. Manages the full pipeline end-to-end.
🔍
functional-reviewer
Opus
Compares the PR diff against acceptance criteria. Finds gaps, risks, and missing coverage.
📝
test-scenario-designer
Sonnet
Generates structured test scenarios: happy path, negative, boundary, and edge cases.
⌨
automation-writer
Sonnet
Converts test scenarios into Playwright, Cypress, or Gherkin code ready to run.
🐛
bug-reporter
Sonnet
Turns review findings into developer-ready bug reports with steps to reproduce.
☑
manual-validator
Sonnet
Guides manual test execution step by step. Tracks pass/fail for each scenario.
⚙
environment-manager
Sonnet
Checks out PR branches, starts the application, and runs health checks.
🌐
browser-validator
Sonnet
Validates scenarios in a real browser via Chrome MCP. Navigates, clicks, verifies.
🚀
release-analyzer
Opus
Multi-repo release diff analysis with cross-repo impact mapping.
🎯
smart-test-selector
Sonnet
Maps code changes to existing tests. Finds what to run and what may break.
